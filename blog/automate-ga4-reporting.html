Automate Weekly Google Analytics Reports to Your Inbox

Gab · March 2026 · 6 min read

Google Analytics 4 (GA4) is powerful, but the interface can be overwhelming. You probably just want to know: "How did we do last week compared to the week before?"

Instead of clicking through menus every Monday morning, why not have a bot email you the key numbers?

The Tools

  • Google Analytics Data API (Beta): Allows you to fetch data programmatically.
  • Python: The language we'll use to script it.
  • GitHub Actions: To run the script on a schedule (for free).

Step 1: Get Access Credentials

This is the hardest part. You need a Service Account JSON key from Google Cloud Console.

  1. Go to Google Cloud Console.
  2. Create a project.
  3. Enable the "Google Analytics Data API".
  4. Create a Service Account and download the JSON key file.
  5. Important: Add the Service Account's email address (client_email in the JSON) as a user in your GA4 property (Admin > Property Access Management). Give it "Viewer" access.

Step 2: Ask Claude for the Script

Now, let Claude do the heavy lifting.

The Prompt:

Write a Python script that uses `google-analytics-data` library to fetch a report for the last 7 days.

I need these metrics:
- Active Users
- Sessions
- Screen Page Views

I need to compare them to the previous 7-day period.

Calculate the percentage change for each.

Format the output as a simple text summary.
I have my credentials in a file called `credentials.json` and my GA4 Property ID in an environment variable `GA4_PROPERTY_ID`.

Claude will give you a script that uses the official client library.

Step 3: Emailing the Report

You can ask Claude to extend the script to send an email using Gmail (via SMTP) or simpler services like Mailgun or Resend.

Extend the script to send this summary via email using `smtplib` and a Gmail App Password.
Env vars: GMAIL_USER, GMAIL_APP_PASSWORD, RECIPIENT_EMAIL.

Step 4: Automate with GitHub Actions

To run this every Monday at 9 AM, use a GitHub Actions workflow with a cron trigger:

name: Weekly Analytics
on:
  schedule:
    - cron: '0 9 * * 1' # Every Monday at 9 AM UTC

jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.9'
      - run: pip install google-analytics-data
      - name: Run Report
        env:
          GA4_PROPERTY_ID: ${{ secrets.GA4_PROPERTY_ID }}
          GMAIL_USER: ${{ secrets.GMAIL_USER }}
          GMAIL_APP_PASSWORD: ${{ secrets.GMAIL_APP_PASSWORD }}
          RECIPIENT_EMAIL: ${{ secrets.RECIPIENT_EMAIL }}
          GOOGLE_APPLICATION_CREDENTIALS_JSON: ${{ secrets.GOOGLE_CREDENTIALS_JSON }}
        run: python report.py

Note: For the credentials JSON, it's best to store the entire JSON string as a GitHub Secret and write it to a file during the workflow run.

The Result

Every Monday, you get an email: "Visits up 15%. Users down 2%." No login required. You know exactly what's happening with your business before your second coffee.

Data without the dashboard fatigue.

Automate your reporting and spend your time acting on data, not finding it.
