How to Automate Stripe Payment Links for Your Micro-SaaS

Gab · March 2026 · 5 min read

Building a custom checkout flow is hard. You have to worry about PCI compliance, tax calculation, responsive design, and error handling.

Stripe Payment Links solve this. You create a link in your Stripe dashboard, send it to a customer (or put it on your landing page), and Stripe handles the rest. But what happens after they pay?

Ideally, you want them to get access to your product immediately — whether it's a software license, a course login, or a download link.

The "Glue" Code: Webhooks

When someone buys via a Payment Link, Stripe fires a "webhook" event (specifically checkout.session.completed). You need a small script to catch this event and do something useful.

You can host this script for free on Vercel, Netlify Functions, or even Google Cloud Functions. And yes, Claude can write it for you.

Step 1: Create the Payment Link

  1. Log in to Stripe.
  2. Go to Products > Add Product.
  3. Click Create payment link next to your new product.
  4. Copy the URL. That's your checkout page.

Step 2: Ask Claude for the Webhook Handler

Here's a prompt to generate a Node.js script for Vercel:

I'm using Stripe Payment Links. I need a Vercel Serverless Function (Node.js) to handle the webhook.
The file should be `api/webhook.js`.

When a `checkout.session.completed` event occurs:
1. Verify the Stripe signature (very important!).
2. Get the customer's email from the session object.
3. Generate a random license key (string of 16 chars).
4. Send an email to the customer with their license key using Resend (or SendGrid).
5. Return a 200 OK status.

I have my STRIPE_WEBHOOK_SECRET and RESEND_API_KEY in environment variables.

Claude will give you the code. You'll need to create a simple package.json with stripe and resend dependencies, which Claude will also provide.

Step 3: Deploy and Connect

  1. Push your code to a GitHub repository.
  2. Connect it to Vercel (free tier).
  3. Add your environment variables in Vercel settings.
  4. Go back to Stripe > Developers > Webhooks.
  5. Add an endpoint: https://your-vercel-project.vercel.app/api/webhook.
  6. Select the checkout.session.completed event.

Result: Automated Sales

Now, whenever someone buys from your link:

  • Stripe charges the card.
  • Stripe tells your Vercel function "Hey, someone paid!"
  • Your function generates a key and emails the customer.

You wake up to money in your bank account and zero support tickets asking "Where is my download?"
