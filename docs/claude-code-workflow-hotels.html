Claude Code Workflow for Hotels

Tool Setup Category A — Hotels
Claude Code operates as the AI pair-programmer that builds and maintains every component of the hotel technology stack — from static site generation to Apps Script automation and CI/CD pipeline configuration. The workflow treats Claude Code as a terminal-native development environment where natural language prompts replace manual coding for HTML scaffolding, webhook handler logic, and deployment scripts. This is the tool that builds the tools.

1.0 Deployment Architecture

Claude Code sits upstream of every other tool in the stack. It is the builder, not the runtime. The development loop for a hotel property:

  1. Operator describes the desired feature or fix in natural language.
  2. Claude Code reads existing project files, understands the architecture, and writes or modifies code.
  3. Changes are committed to the GitHub repository.
  4. GitHub Actions (or GitHub Pages auto-deploy) pushes the site live.
  5. Backend logic (Apps Script) is updated via clasp push.

2.0 Prerequisites

Installation

bash# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Verify installation
claude --version

# Launch in your project directory
cd ~/projects/auberge-site
claude

Authentication

Claude Code requires either:

  • Anthropic Max plan — direct authentication via browser login.
  • API key — set ANTHROPIC_API_KEY environment variable.
bash# API key method
export ANTHROPIC_API_KEY="sk-ant-xxxxx"

# Or add to shell profile for persistence
echo 'export ANTHROPIC_API_KEY="sk-ant-xxxxx"' >> ~/.zshrc

Project Configuration

Create a CLAUDE.md at the project root. This file persists context across sessions:

markdown# CLAUDE.md — Auberge de nos Aieux

## Project
Static hotel website deployed on GitHub Pages.
Payment processing via Stripe Payment Links.
Backend automation via Google Apps Script.

## Stack
- HTML/CSS/JS (no framework)
- Stripe Payment Links + Webhooks
- Google Apps Script (clasp for local dev)
- WhatsApp Business API (Cloud API)
- GitHub Pages (auto-deploy from main branch)

## Conventions
- CSS uses custom properties defined in styles.css
- All pages use Inter font family
- Dark theme: --bg: #0A0A0B, --accent: #6C5CE7
- Filenames: lowercase-kebab-case.html
- No build step. No bundler. Raw HTML.

## Important Paths
- /index.html — Main landing page
- /rooms/ — Individual room pages
- /scripts/ — Apps Script source (synced via clasp)
- /styles.css — Global design system

3.0 Step-by-Step Configuration

3.1 Generating a Room Page

bash# In Claude Code session:
> Create a new room page for "Chambre Forêt" at /rooms/chambre-foret.html.
> It should follow the same structure as chambre-riviere.html.
> Queen bed, forest view, sleeps 2. Rate: $180/night.
> Include a Stripe Payment Link button pointing to
> https://buy.stripe.com/xxxx
> Add OG meta tags and structured data for SEO.

Claude Code reads the existing room page, replicates the pattern, and writes the new file with correct metadata, pricing, and CTA links.

3.2 Writing a Webhook Handler

bash# In Claude Code session:
> Read the current Apps Script files in /scripts/.
> Add a handler for the Stripe charge.refunded event.
> It should update the booking status to "refunded" in the
> Bookings sheet and send a WhatsApp message to the guest
> confirming the refund amount.

3.3 Bulk Operations

bash# Update pricing across all room pages
> Update the nightly rate for all rooms in /rooms/ by 15%.
> Update the corresponding Stripe product metadata.
> Regenerate the pricing table on index.html.

# Generate seasonal content
> Create a "Winter Packages" section on index.html with
> 3 package cards. Each links to a different Stripe
> Payment Link. Use the existing card component style.

3.4 Git Workflow Integration

Claude Code handles the full git cycle:

bash# After making changes:
> /commit

# Claude Code will:
# 1. Run git status to see changes
# 2. Stage relevant files
# 3. Write a descriptive commit message
# 4. Create the commit

# To push:
> Push to origin main

# Claude Code respects branch protection and will
# create PRs if direct push is not allowed.

3.5 Memory Across Sessions

Claude Code persists project knowledge via:

  • CLAUDE.md — project-level context (conventions, paths, stack).
  • ~/.claude/memory/ — cross-project learnings and patterns.
  • Auto-memory — Claude Code automatically records debugging insights and patterns.
Treat CLAUDE.md like infrastructure. Keep it updated. A well-maintained CLAUDE.md means Claude Code starts every session with full context — no ramp-up time.

4.0 Integration Points

GitHub Pages

Claude Code writes HTML files and pushes to GitHub. GitHub Pages auto-deploys. The entire site build-test-deploy cycle happens in the terminal.

Google Apps Script

Claude Code writes .gs files locally and deploys via clasp. Webhook handlers, trigger functions, and sheet operations are all authored through prompts.

Stripe Integration

Claude Code generates Payment Link embeds, writes webhook consumer code, and updates product metadata via the Stripe CLI.

WhatsApp Automation

Claude Code scaffolds WhatsApp template message functions and iterates on message copy and trigger timing logic.
