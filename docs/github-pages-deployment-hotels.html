GitHub Pages Deployment for Hotels

Tool Setup Category A — Hotels
GitHub Pages provides zero-cost, zero-maintenance hosting for hotel property websites by serving static HTML directly from a Git repository with automatic SSL and CDN distribution. The deployment model eliminates hosting fees, server administration, and WordPress vulnerabilities by treating the entire website as a Git repository that Claude Code can read, modify, and deploy in a single terminal session. Combined with Stripe Payment Links and a Google Apps Script backend, this creates a full hotel web presence for $0/month in hosting costs.

1.0 Deployment Architecture

GitHub Pages sits at the front of the stack. It is the only public-facing surface. Everything behind it — Stripe, Apps Script, WhatsApp — operates through API calls and webhooks that the static site itself never handles.

3.0 Step-by-Step Configuration

3.1 Create Repository

bash# Create and initialize
mkdir auberge-site && cd auberge-site
git init

# Create the repository on GitHub
gh repo create auberge-de-nos-aieux --public --source=. --push

# Or for an organization:
gh repo create GabHotels/auberge-site --public --source=. --push

3.2 Enable GitHub Pages

bash# Enable Pages via CLI (deploy from main branch, root directory)
gh api repos/{owner}/{repo}/pages \
  --method POST \
  --field source='{"branch":"main","path":"/"}'

# Or via GitHub UI:
# Settings → Pages → Source: Deploy from branch → main → / (root)

3.3 Configure Custom Domain

bash# Create CNAME file in repo root
echo "aubergedenosai eux.com" > CNAME

# DNS configuration at your registrar:
# A records (GitHub Pages IPs):
#   185.199.108.153
#   185.199.109.153
#   185.199.110.153
#   185.199.111.153
#
# CNAME record (for www subdomain):
#   www → your-username.github.io

# Verify DNS propagation
dig aubergedenosai eux.com +short

# Enable HTTPS (automatic after DNS propagates)
gh api repos/{owner}/{repo}/pages \
  --method PUT \
  --field https_enforced=true

3.5 GitHub Actions for Validation (Optional)

Add a CI step that validates HTML and checks for broken links before deploy:

yaml# .github/workflows/validate.yml
name: Validate HTML

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Check HTML
        uses: anishathalye/proof-html@v2
        with:
          directory: .
          check_html: true
          check_opengraph: true
          enforce_https: true

      - name: Check links
        uses: lycheeverse/lychee-action@v1
        with:
          args: --exclude-loopback --no-progress './**/*.html'
          fail: true

4.0 Integration Points

Stripe Integration

Payment Link URLs are embedded as href attributes on CTA buttons throughout the static site. No Stripe.js needed — checkout is fully Stripe-hosted.

Google Apps Script

The static site fetches room availability from the Apps Script doGet() endpoint. This is the only dynamic data on an otherwise static site.

Claude Code

Claude Code is the site builder. It generates pages, updates content, and pushes deployments — all from the terminal. The entire site is maintained through prompts.

WhatsApp Automation

The static site includes click-to-chat links (wa.me/number) for pre-booking inquiries. Post-booking communication happens entirely off-site via the WhatsApp API.
